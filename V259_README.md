# V259 前台顯示隔離修正版

目的：解決 01/02 進頁後整頁顯示完成仍需 30~60 秒的問題。

## 修改檔案
- pages/01_01. 工時紀錄.py
- pages/02_02. 歷史紀錄.py
- pages/99_99. 效能診斷.py

## 01. 工時紀錄
- 開始作業區不再於每次選單變更時預先查 active/duplicate/conflict。
- duplicate/conflict 檢查改到按下「開始作業」後才執行，確保資料正確但不拖慢整頁顯示。
- 結束作業區移除每次 render 的 heavy refresh，只保留直接 active 查詢。
- 今日已結束紀錄改成手動「重新整理已結束紀錄」。
- 今日工時明細改成手動「重新整理今日明細」，並使用 session 快取。
- 按開始/暫停/完工/下班後清除顯示快取，但不強迫整頁立即重查重表格。

## 02. 歷史紀錄
- 開頁不自動查歷史明細。
- 套用篩選或按「查詢 / 重新整理歷史明細」才查資料。
- 歷史明細使用 session 快取，避免每次 rerun 重查。

## 權限與診斷
- 99. 效能診斷使用正確檔名：pages/99_99. 效能診斷.py。

## 注意
若 GitHub 內仍存在舊的 mojibake 頁面檔名（例如 #U5de5...），Streamlit 可能同時顯示舊頁與新頁。正式合併時請在 GitHub 刪除舊 mojibake 頁面檔，只保留正常中文檔名。
